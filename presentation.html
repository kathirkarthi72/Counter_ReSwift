Core Concept

What is ReSwift?

ReSwift is a Redux implementation for Swift that enforces a strict Unidirectional Data Flow.

📦 1. AppState

A single, immutable state tree storing all application data. Read-only from views.

⚡ 2. Actions

Declarative value structs/enums describing user or system intent to mutate state.

⚙️ 3. Reducers

Pure functions (Action, State?) -> State that compute new state without side-effects.

Middleware

Handling Side-Effects with ReSwiftThunk

Reducers must remain pure and cannot execute asynchronous tasks (URLSession, DB reads, timers). ReSwiftThunk solves this cleanly.

Complete Application Architecture

Sync Action Flow

View ➔ dispatch(CounterAction.increment) ➔ counterReducer() ➔ AppState updated ➔ SwiftUI View updates

Async Thunk Flow

View ➔ dispatch(fetchPostsThunk()) ➔ Thunk Middleware ➔ PostService API ➔ dispatch(fetchSuccess) ➔ postsReducer() ➔ View updates

SwiftUI Integration (ObservableStore)

Conforms to StoreSubscriber and ObservableObject. Converts ReSwift state changes into @Published updates on the @MainActor.

Code Walkthrough

ReSwiftThunk Implementation

public func fetchPostsThunk(service: PostServiceProtocol = PostService.shared) -> Thunk<AppState> {
    return Thunk<AppState> { dispatch, getState in
        // 1. Notify store that loading started
        dispatch(PostsAction.fetchStarted)
        
        Task {
            do {
                // 2. Perform async network request
                let posts = try await service.fetchPosts()
                
                await MainActor.run {
                    // 3. Dispatch success pure action
                    dispatch(PostsAction.fetchSuccess(posts))
                }
            } catch {
                await MainActor.run {
                    dispatch(PostsAction.fetchFailed(error.localizedDescription))
                }
            }
        }
    }
}
Best Practices

Key Takeaways for Developers

✅ Do's
  • Keep Reducers 100% pure functions.
  • Use Thunks for networking, timers, and disk reads.
  • Decompose state into sub-states (CounterState, PostsState).
  • Ensure MainActor updates for SwiftUI rendering.
❌ Don'ts
  • Never make API calls inside Reducers.
  • Never mutate state directly outside Reducers.
  • Avoid duplicating data across sub-states.
